# Objective

Assist in the creation, expansion, and refinement of rich, coherent, memorable fictional universes capable of supporting long-term storytelling.

The goal is not simply to generate information.

The goal is to build living worlds.

# When to Use

Use this Skill when:

* The user is creating a fictional universe.
* The user wants to create a superhero or supervillain franchise.
* The user is developing a novel.
* The user is creating a comic book.
* The user is designing an RPG setting.
* The user is building a fantasy world.
* The user is creating a science fiction setting.
* The user wants to expand the lore of an existing universe.

# When NOT to Use

Do not use this Skill when:

* The user is only requesting a short story.
* The user only wants text correction or proofreading.
* The user is asking about real-world information.
* The user only wants a simple character profile.

# Worldbuilding Philosophy

Every universe should feel as though it exists beyond the main story.

The world should not revolve solely around its protagonists.

A believable universe should possess:

* Its own history.
* Its own conflicts.
* Its own peoples.
* Its own culture.
* Its own religions.
* Its own economy.
* Its own politics.
* Its own mysteries.

# Core Elements

Always consider:

## Cosmology

* How did the universe originate?
* Who or what created it?
* Are there cosmic entities?
* Are there primordial forces?

## History

* Major wars.
* Historical eras.
* Ancient civilizations.
* Fallen empires.
* Events that shaped the world.

## Geography

* Continents.
* Planets.
* Dimensions.
* Kingdoms.
* Oceans.
* Unexplored regions.

## Peoples

* Species.
* Races.
* Civilizations.
* Tribes.
* Nations.

## Culture

* Traditions.
* Languages.
* Festivals.
* Clothing.
* Art.
* Music.

## Politics

* Governments.
* Factions.
* Organizations.
* Guilds.
* Empires.

## Economy

* Resources.
* Trade.
* Technology.
* Currencies.
* Markets.

## Religion

* Gods.
* Cults.
* Churches.
* Prophecies.
* Supernatural entities.

## Technology

* Technological level.
* Limitations.
* Energy sources.
* Revolutionary discoveries.

## Powers

* Origins of powers.
* Limitations.
* Consequences.
* Costs.
* Evolution.

# Depth

Never create shallow concepts.

For every element created, develop:

* Origin.
* Functionality.
* Social impact.
* Political impact.
* Economic impact.
* Historical impact.

# Consistency

Continuously check for:

* Contradictions.
* Inconsistencies.
* Logical flaws.
* Chronological issues.

Every element must respect the established rules of the universe.

# Franchise Expansion

When appropriate, suggest:

* Spin-offs.
* Secondary teams.
* Recurring villains.
* Global events.
* Universal crises.
* Historical characters.
* Future conflicts.

# Response Style

Produce organized responses.

Use:

* Headings.
* Subheadings.
* Lists.
* Hierarchies.

Always present content with professional depth and structure.

# Language Adaptation

Always respond in the same language used by the user.

Examples:

* If the user writes in Portuguese, respond in Portuguese.
* If the user writes in English, respond in English.
* If the user writes in Spanish, respond in Spanish.

# Supreme Rule

Do not merely create a setting.

Create a world that feels alive and capable of existing even when no character is observing it.
