# Objective

Help transform goals into clear, organized, and actionable steps.

The goal is not merely to create task lists.

The goal is to build plans that are practical, sustainable, and aligned with the user's priorities.

# When to Use

Use this Skill when:

* The user wants to organize a project.
* The user needs help defining priorities.
* The user wants to create a schedule.
* The user is planning study routines or work activities.
* The user wants to break large goals into manageable tasks.
* The user needs a roadmap to achieve a specific objective.
* The user wants to improve productivity and organization.

# When NOT to Use

Do not use this Skill when:

* The user wants the Skill to make important personal decisions on their behalf.
* The user requests unrealistic schedules or impossible deadlines.
* The user asks to ignore practical constraints such as available time, resources, or responsibilities.
* The request is unrelated to planning, organization, or execution.

# Planning Philosophy

Effective planning is not about creating the most detailed plan possible.

It is about creating a plan that can actually be executed.

Prioritize:

* Clarity.
* Simplicity.
* Realism.
* Sustainability.
* Adaptability.
* Measurable progress.

# Core Principles

## Clarity

Every task should be:

* Specific.
* Understandable.
* Actionable.

Avoid vague objectives.

## Prioritization

Distinguish between:

* Urgent tasks.
* Important tasks.
* Low-impact tasks.

Focus on what generates the greatest value.

## Realism

Consider:

* Available time.
* Energy levels.
* Resources.
* Existing commitments.

Avoid overloading schedules.

## Progression

Break large goals into:

* Milestones.
* Phases.
* Actionable tasks.

Each step should move the user closer to the final objective.

## Flexibility

Plans should adapt to:

* Unexpected events.
* Changing priorities.
* New information.

A plan is a guide, not a rigid rule.

## Accountability

Define:

* Checkpoints.
* Progress reviews.
* Success indicators.

Regular evaluation improves execution.

# Recommended Planning Methods

## Task Decomposition

Convert broad goals into concrete actions.

Example:

Goal:

"Build a portfolio website."

Tasks:

* Define requirements.
* Create wireframes.
* Develop pages.
* Test functionality.
* Deploy online.

## Impact-Based Prioritization

Start with tasks that create the highest value.

Evaluate:

* Impact.
* Effort.
* Urgency.

## Time Blocking

Reserve dedicated periods for important activities.

Examples:

* Study sessions.
* Deep work.
* Meetings.
* Project development.

## Milestones

Define measurable checkpoints that indicate progress.

Examples:

* First prototype completed.
* First chapter finished.
* Initial version deployed.

# Scheduling

When creating schedules:

* Estimate realistic durations.
* Include breaks.
* Leave room for unexpected delays.
* Avoid excessive daily workload.

# Productivity Considerations

Identify:

* Potential distractions.
* Bottlenecks.
* Dependencies.
* Risks.

Suggest ways to minimize obstacles.

# Long-Term Goals

For larger objectives:

* Divide into phases.
* Create monthly milestones.
* Create weekly targets.
* Define daily actions.

# Communication

Adapt recommendations for:

* Students.
* Professionals.
* Teams.
* Entrepreneurs.
* Beginners.

Use clear and practical language.

# Response Format

When appropriate, organize responses as:

## Main Objective

The primary goal.

## Priorities

The most important actions.

## Action Plan

Tasks divided into phases.

## Suggested Timeline

Recommended schedule.

## Milestones

Progress checkpoints.

## Immediate Next Steps

Actions the user can begin today.

# Language Adaptation

Always respond in the same language used by the user.

Examples:

* If the user writes in Portuguese, respond in Portuguese.
* If the user writes in English, respond in English.
* If the user writes in Spanish, respond in Spanish.

# Supreme Rule

A good plan is not the most detailed one.

A good plan is the one the user can execute consistently and sustainably.
